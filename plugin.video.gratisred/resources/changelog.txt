[B]- 1.0.8 (2026-04-29)[/B] FlareSolverr cookie cache + new TV providers:

Both [I]bstsrs.in[/I] AND [I]srstop.link[/I] are CF-protected, so each scraper was round-tripping through FlareSolverr for every candidate URL (~15–30s per call × 5–7 URLs). [B]- 1.0.7 (2026-04-29)[/B] Cloudflare auto-bypass via FlareSolverr (opt-in):
 * Why bstsrs went silent: [I]bstsrs.in[/I] was placed behind a Cloudflare managed JS challenge ([I]cf-mitigated: challenge[/I]) before v1.0.5; it was not srstop that broke it. Plain [I]requests[/I] (what Kodi uses) gets a 403 "Just a moment..." page on every URL, so the regex finds zero [I]dbneg(...)[/I] embeds and the scraper returns []. The two alternate domains are also gone: [I]bstsrs.one[/I] 302s to an ad redirector and [I]bstsrs.cc[/I] 302s to a parking page ([I]ww1.bstsrs.cc[/I]). You only noticed once srstop began contributing ~19 links per episode and the bstsrs absence became conspicuous.
 * Cloudflare auto-bypass via FlareSolverr (NEW, opt-in):
    - [I]resources/lib/modules/client.py[/I]: [I]scrapePage()[/I] now detects CF managed-challenge / IUAM responses (status 403/503 with [I]cf-mitigated: challenge[/I] or "Just a moment..." body) and transparently retries the same request through a self-hosted FlareSolverr instance. Returns a duck-typed Response so all existing callers keep working.
    - [I]resources/settings.xml[/I]: NEW field under Playback Settings — [B]FlareSolverr URL[/B]. Blank by default => behaviour identical to v1.0.6. Set it to e.g. [I]http://127.0.0.1:8191/v1[/I] to unlock bstsrs and any future CF-protected provider.
 * bstsrs scraper hardening continued:
    - Drops the dead [I]bstsrs.one[/I] and [I]bstsrs.cc[/I] fall-backs (they were just costing 30+ seconds of timeouts per episode for nothing).
    - Tries the Italian local title first via [I]localtvshowtitle[/I] ([I]bstsrs.in[/I] URLs use Italian show names, e.g. "House M.D." -> "/show/dr-house-medical-division-..."). Then falls back to the original title slugs and the [I]/ajax/search.php?q=[/I] search API for both titles.
    - Short-circuits as soon as a CF challenge is detected when no FlareSolverr URL is configured, so it no longer slows the overall episode scrape down.
 * How to enable bstsrs:
    - Run FlareSolverr on the same machine as Kodi (Docker: [I]docker run -d -p 8191:8191 --restart unless-stopped --name flaresolverr ghcr.io/flaresolverr/flaresolverr:latest[/I]; Windows/Linux/macOS prebuilt binaries: [I]https://github.com/FlareSolverr/FlareSolverr/releases[/I]).
    - In Kodi: Gratis Red > Settings > Playback Settings > scroll to [B]Cloudflare Bypass (FlareSolverr)[/B] > paste [I]http://127.0.0.1:8191/v1[/I].
    - Re-open any episode — [I]bstsrs.in[/I] entries will now appear alongside [I]srstop.link[/I].

    - Wrong network IDs in [I]resources/lib/indexers/metadata/tvmaze/tvmaze_networks.py[/I] — the curated list shipped with v1.0.2 had several IDs pointing at the wrong network on TVmaze (e.g. FX -> Cinemax, BBC Two -> FX, FXX -> ITV1, Channel 4 -> BBC America, BBC Three -> TNT, Channel 5 -> Syfy, Disney XD -> ITV2, History (US) -> History (CA), Sky Atlantic -> SundanceTV). We now build a deduplicated, cached "all shows" pool from [I]/shows?page=0..N[/I] (parallel page fetches, ~25s on first build, instant thereafter for the cache TTL) and filter against that pool.
